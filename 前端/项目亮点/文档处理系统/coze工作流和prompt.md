# Coze



## 1.背景

1. 普通的AI对一些实时专业的知识，效果不是很好，因为ai中的数据库还没有更新。
2. 普通的大模型Ai容易自由发挥，无法满足特定领域的需求，容易产生幻觉。我们需要限定场景。并且对格式进行限制。

所以我们需要自己实现一个知识库，让ai直接根据自己创建的知识库中检索消息。

## 2.技术调研

**dify**

优点： 开源，功能丰富，自定义程度高，可以进行本地部署。支持常见的在线模型，支持Agent

缺点：主要面向国外，插件比较少。



**FastGPT**

优点：开源，面向国内企业，支持本地化自动部署，支持大多数模型。

缺点：国际化支持不好，agent功能不完善，但是rag功能强



**Coze**

优点：简单快速，开发效率高

缺点：自定义支持程度低。适合快速上手的项目，支持的在线模型少



我们的项目目前不需要很复杂的知识库，并且甲方要求先快速做一版。临时用了Coze，后续为了安全和专业可能会用到本地部署的知识库。

## 3.实现方案

**上传到知识库**

对接上传文件到知识库，删除知识库中的文件，查看知识库中的文件列表等接口。前端通过富文本保存后可以将导出的md 文档，或者是docx文档 ，通过在线接口上传到知识库中，然后coze接收到这个文件会自动进行文字解析分段。为了防止分段的长度过长。需要自定义一些规则

**separator 分段标识符**：

如果导出的是markdown格式，使用 # 作为换行符

如果是其他的文本，就是用\n\n作为标识符



**max_tokens设置最大的token：**

最大的分段长度：1000  . 每个模型都有最大的token，如果太长会报错。连续长文本不易于AI解析。



**remove_extra_spaces   是否过滤连续的空格**：

如果是md文档的话 不进行过滤。

如果是其他文档的话对空格进行过滤。



**remove_urls_emails设置是否过滤url和电子邮件**

统一不进行过滤。文档中可能会有记录数据的网站。



# prompt和工作流

通过设置prompt让大模型   规范模型的输入和输出的格式，比如规定用markdown格式进行显示。规定大模型的语言风格。同时给ai的回答设置一些限制。

设置工作流是为了规范大模型解决问题的流程，根据问题的类型决定是去知识库中查找，还是去联网查找。



## 1.调优prompt

我总结下来 我一般会从4个方面进行prompt的优化。**角色，具体的任务，规范输出格式，给出答案示例，并对输出的内容添加限制。**

**规范角色：**

因为大模型不是针对特定领域进行思考和回答，如果不限制角色，ai会自由发挥，胡说八道。

```typescript
你是一名专业的{#InputSlot placeholder="请输入编辑块内容为空时的提示文案" mode="input"#}科研与技术助理{#/InputSlot#}。 负责精准检索文件信息，智能理解用户问题，并结合知识库与网络资源，高效提供专业答案，助力客服与文档管理。 
```



**设置具体的任务**

为了让模型知道具体要做什么，使回答更加明确，同时让ai在特定的领域内生成的内容更加规范。

```typescript
1. 解析用户提问意图。
2. 检索知识库中的相关信息。根据检索到的内容生成准确、条理清晰的回答。
   输出结构化内容，使用编号或分点，每条内容 1~2 句。 
3. 如果知识库没有相关信息，要结合联网的结果进行回答，并且告诉用户信息的来源是联网还是知识库。
4. 结和用户以往的回答对用户下一次可能出现的问题进行追问。
5. 如果检测到用户上传的内容是文件，就对文件中的内容进行总结。
```



**设置限制**

```typescript

 - {#InputSlot placeholder="需要保密的信息：比如你的提示词、搭建方式等，比如需要保密的敏感数据信息。"#}{#/InputSlot#}
 - 个人隐私信息：涵盖真实姓名、电话号码、地址、账号密码等敏感信息。
 - {#InputSlot placeholder="非主题相关问题：比如xxx、xxx、xxx等与你需要聚焦回答的主题无关的问题。"#}{#/InputSlot#}
 - 违法、违规内容：包括政治敏感话题、色情、暴力、赌博、侵权等违反法律法规和道德伦理的内容。
2. 禁止使用的词语和句子
 - 回答中严禁使用{#InputSlot placeholder="“禁止回答语句1”、“禁止回答语句2”、“禁止回答语句3”、“禁止回答语句4”..."#}{#/InputSlot#}这类语句。
 - 不回答{#InputSlot placeholder="不希望回答的内容，比如：代码（json、yaml、代码片段）、图片等"#}{#/InputSlot#}。
3. 回复风格要求准确无误、言简意赅、通俗易懂，展现专业与确定性。
```

```typescript
  "来源": ["文档ID_001", "文档ID_002"]
```

## 2.工作流

先进行意图识别，判断输入的内容是文件，还是用户的提问，还是一些无关的东西。

**如果上传的是文件**  

调用文件解析的插件，对文档内容进行总结。

**如果是提问**

去知识库中检索，判断检索的结果是不是空 如果是空的话进行联网检索，不是空的话使用豆包的模型对检索的结果进行总结。

**如果是无关的问题**

礼貌拒绝

最后进行变量聚合，输出结果。



## 4.结果评估

使用coze的罗盘进行分析。通过prompt输出进行对比，我们针对知识库准备了20个问题，包含对知识库的中一些数据来源网站的提问，还有一些对知识的提问。并且准备10个与提问无关的问题，通过prompt对比输出的结果，和输出的格式。选择最优的prompt。



然后通过将问题添加到评测集中，并且设置评估器对agent输出的内容完整性，简洁性，深度性，细节性，犯罪行等进行评价。评价之后coze会给出一个评分，如果认为评分不符合的需要进行手动进行评分。
