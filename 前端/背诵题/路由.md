| 对比       | Hash 路由    | History 路由 |
| ---------- | ------------ | ------------ |
| URL形式    | `/#/home`    | `/home`      |
| 实现原理   | hashchange   | pushState    |
| 服务器请求 | 不会重新请求 | 可能重新请求 |
| SEO        | 不友好       | 友好         |
| 兼容性     | 非常好       | IE10+        |
| 服务器配置 | 不需要       | 需要         |

Hash 路由是通过 URL 中的 # 片段实现的，
通过监听 hashchange 事件来实现页面切换，
因为 # 后面的内容不会发送到服务器，所以不会触发页面刷新，
兼容性很好，但 URL 不够美观，SEO 不友好。





History 路由是基于 HTML5 History API 实现的，
通过 pushState 和 replaceState 修改 URL，
并监听 popstate 事件处理浏览器前进后退，
URL 更加美观，SEO 更友好，但刷新时会请求服务器，
因此需要服务器配置将所有路径重定向到 index.html。





## history

在浏览器内部维护一个 **历史记录栈**

pushState 就是向栈中添加一条记录，修改url，页面不会刷新。

replaceState 就是替换历史记录。

popstate 当浏览器的历史记录发生变化的时候触发
