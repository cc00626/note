### 受控组件和非受控组件

1. **受控组件 (Controlled Components)**

**核心定义：** 组件的表单数据由 React 组件的 **state** 处理。React 是唯一的“真理来源（Single Source of Truth）”。

**工作原理**：

1. **绑定：** 表单的 `value` 绑定到一个 `state`。
2. **监听：** 通过 `onChange` 事件监听用户输入。
3. **反馈：** 在事件回调中通过 `setState` 更新 state，从而触发 Fiber 的 Render 阶段，重新渲染组件。

JavaScript

```
const ControlledInput = () => {
  const [value, setValue] = useState('');
  return <input value={value} onChange={(e) => setValue(e.target.value)} />;
};
```

**优点**：

- **即时验证：** 可以在用户输入的每一秒进行校验（比如手机号长度、实时搜索）。
- **强制格式：** 比如用户输入小写，你可以实时将其转为大写。
- **UI 联动：** 输入框的值可以直接决定页面上其他组件的显示状态。

------

**2. 非受控组件 (Uncontrolled Components)**

**核心定义：** 表单数据由 **DOM 节点自身** 处理。React 不跟踪其状态变化。

**工作原理：**

1. **挂载：** 使用 `useRef` 获取 DOM 节点的引用。
2. **提取：** 在需要数据时（比如点击提交按钮），通过 `ref.current.value` 直接从 DOM 中“拽”出数据。
3. **Fiber 角色：** 此时数据存在 DOM 节点（即 Fiber 对象的 `stateNode`）中，而不是 `memoizedState` 里。

JavaScript

```
const UncontrolledInput = () => {
  const inputRef = useRef(null);
  const handleSubmit = () => console.log(inputRef.current.value);
  return <input ref={inputRef} defaultValue="初始值" />;
};
```

**优点：**

- **代码简洁：** 不需要写繁琐的 `onChange` 逻辑。
- **性能略高：** 因为用户输入不会频繁触发 React 的 Render 阶段。
- **集成方便：** 容易集成非 React 的第三方 DOM 库。
