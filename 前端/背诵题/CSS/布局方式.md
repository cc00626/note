### Positon布局

1. `static` (静态定位)

- **默认值**：所有元素默认都是 `static`。
- **表现**：元素遵循**标准文档流**，即块级元素垂直排列，行内元素水平排列。
- **注意**：设置 `top`、`left`、`z-index` 等属性均**无效**。

2. `relative` (相对定位)

- **参照物**：元素**自身在文档流中的原始位置**。
- **表现**：虽然可以通过 `top`、`left` 偏移，但元素在文档流中**原本占据的空间依然保留**（不会导致后面元素上移）。
- **核心用途**：
  1. 进行微调。
  2. **作为绝对定位（absolute）子元素的参照基点**（子绝父相）。

3. `absolute` (绝对定位)

- **参照物**：最近的一个**非 static 定位**的祖先元素。如果一直往上找都没找到，就参照 `body`（浏览器视口）。
- **表现**：**完全脱离文档流**，不再占据原有空间。
- **注意**：绝对定位的元素会变成类似块级元素的性质（如果没有设置宽度，宽度由内容撑开）。

4. `fixed` (固定定位)

- **参照物**：**浏览器窗口 (Viewport)**。
- **表现**：**完全脱离文档流**，且不随滚动条滚动。无论页面怎么滑，它都钉在屏幕的某个位置。
- **典型场景**：返回顶部按钮、固定顶栏、侧边悬浮广告。

5. `sticky` (粘性定位)

- **表现**：它是 `relative` 和 `fixed` 的结合体。在跨越特定阈值前，它表现得像 `relative`；当页面滚动到特定位置（如 `top: 0`）时，它会像 `fixed` 一样“粘”在顶部。
- **前提条件**：
  1. 必须配合 `top`、`bottom`、`left` 或 `right` 使用。
  2. 父元素不能设置 `overflow: hidden`，否则粘性会失效。
- **典型场景**：表格的标题行、通讯录的字母标题。

### flex布局

作用在父元素

| **属性**              | **作用**                       | **常用值**                                           |
| --------------------- | ------------------------------ | ---------------------------------------------------- |
| **`flex-direction`**  | 决定主轴的方向                 | `row` (默认), `column` (纵向), `row-reverse`         |
| **`flex-wrap`**       | 换行规则                       | `nowrap` (默认，强行挤在一行), `wrap` (换行)         |
| **`flex-flow`**       | 简写属性                       | 是 `direction` 和 `wrap` 的合并写法                  |
| **`justify-content`** | **主轴**上的对齐方式           | `center`, `space-between` (两端对齐), `space-around` |
| **`align-items`**     | **交叉轴**上的对齐方式（单行） | `center`, `flex-start`, `stretch` (拉伸占满)         |
| **`align-content`**   | **多根轴线**的对齐方式         | 仅在有换行（`wrap`）且多行时生效                     |

作用在子元素

**`flex-grow` (放大比例)**：

- 默认为 `0`（即便有空间也不放大）。
- 设为 `1`：均分剩余空间。

**`flex-shrink` (缩小比例)**：

- 默认为 `1`（空间不足时自动缩小）。
- 设为 `0`：即使空间不足也不缩小，常用于固定侧边栏。

**`flex-basis` (基准大小)**：

- 在分配空间前，项目占据的主轴空间。默认 `auto`（即元素本来的宽度）。

**`flex` (核心简写)**：

- **面试必问：`flex: 1` 是什么的缩写？**
- 它是 `flex-grow: 1; flex-shrink: 1; flex-basis: 0%;` 的简写。

**`align-self`**：

- 允许单个项目有与其它项目不一样的对齐方式，可覆盖父元素的 `align-items`。

**`order`**：

- 定义排序顺序。数值越小，越靠前。

### BFC

bfc是块级格式化文本上下文,是在页面中独立渲染的一块区域，不受其他区域的影响。

**特点：**

1. BFC中的块级元素上下margin会重叠，会取两者间最大的。
2. BFC中的元素不会和浮动元素重叠，会计算浮动元素的高度，解决浮动问题。
3.  每个元素的左外边距与包含块的左边界相接触（从左向右），即使浮动元素也是如此 。

**如何开启**

- **根元素** (`<html>`)。

- **浮动元素**：`float` 不为 `none`。

- **绝对定位元素**：`position` 为 `absolute` 或 `fixed`。

- **行内块元素**：`display: inline-block`。

- **表格单元格或标题**：`display: table-cell` 或 `table-caption`。

- **overflow 溢出隐藏**：`overflow` 不为 `visible`（如 `hidden`, `auto`, `scroll`）。

- **弹性/网格容器**：`display: flex`, `inline-flex`, `grid`, `inline-grid`。

### 清除浮动

1. 额外标签法

在元素内部下方添加一个空的标签，css设置成clear：both

```html
<div class="father">
  <div class="box">box1</div>
  <div class="box">box2</div>
  <div class="box">box3</div>
  <div style="clear: both;"></div>
</div>
```

2. 开启BFC

设置overflow：hidden ，缺点 ：会裁剪掉溢出的部分。

3. after为元素

```css
.clearfix:after{
//相当于设置一个块级伪元素
    content: "";
    display: block;
    height: 0;
    //设置clear: both
    clear: both;
    visibility: hidden;
}
```

### 水平居中

1. 绝对定位

   ```css
   .parent { position: relative; }
   .child {
     position: absolute;
     top: 50%;
     left: 50%;
     transform: translate(-50%, -50%);
   }
   ```

2. flex布局

   ```css
   .parent {
     display: flex;
     justify-content: center; /* 水平居中 */
     align-items: center;     /* 垂直居中 */
   }
   ```

3.  Grid 布局

   ```css
   .parent {
     display: grid;
     place-items: center;
   }
   ```

4.  绝对定位 + `margin: auto`

   ```
   .child {
     position: absolute;
     top: 0; left: 0; right: 0; bottom: 0;
     margin: auto;
     width: 200px;  /* 必须定宽 */
     height: 100px; /* 必须定高 */
   }
   ```

5. 行内块 `table-cell` 方式

   ```
   .parent {
     display: table-cell;
     vertical-align: middle; /* 垂直居中 */
     text-align: center;    /* 水平居中 */
   }
   ```

### 文档流

文档流是浏览器在排列网页元素时，遵循的一种**从左到右、从上到下**的布局顺序。

“脱离文档流”是指某个元素从正常的排版规则中**“抽离”**了出来。

一旦脱离文档流，该元素就变得像“幽灵”一样：

1. **不占位**：它在原先文档流中的空间会被后面紧跟的元素**直接占据**（后面的元素会向上平移，填补空隙）。
2. **漂浮感**：它会覆盖在其他元素之上，或者根据特定的参照物进行定位。
