### tailwind css

**解决问题**

1. 起名问题
2. 维护困难，css只增不减
3. 上下文切换



**解决办法**：原子化和局部性



**争议**

1. 类名太长，太丑了
2. 需要额外的学习



**🥰 谁最受益？**

- **全栈/独立开发者：** 无需设计基础，也能快速搞定“不难看”的界面。
- **长期维护的大型团队：** 消灭代码风格差异。不再争论“是 15px 还是 16px”，只有 `m-4`。
- **设计系统导向：** 配置文件 (Config) 成为单一事实来源。



**😫 谁会痛苦？**

- **Pixel-Perfect 还原主义者：** 需要大量 `w-[13px]` 这种任意值，写起来比内联样式还累。
- **极简静态页面的初学者：** 写个 10 行 CSS 的 Demo 还要配 Node 环境，简直杀鸡用牛刀。
