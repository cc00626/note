### 响应式原理和区别

**Vue 2 响应式：基于 `Object.defineProperty`**

**核心机制：数据劫持 + 观察者模式**

Vue 2 的响应式是一个典型的“预处理”过程。

- **初始化（Observe）**：Vue 在创建实例时，会调用 `observe` 方法遍历 `data`。它通过 `Object.defineProperty` 将属性转化为 getter/setter。
- **依赖收集（Getter）**：每个组件实例对应一个 **Watcher** 实例。渲染时，访问属性触发 getter，将当前 Watcher 收集到该属性的 **Dep**（依赖管理器）中。
- **派发更新（Setter）**：当修改属性时，触发 setter。Dep 会通知所有收集到的 Watcher，Watcher 异步执行 `run` 方法触发组件重新渲染。

**存在的缺陷：**

1. **无法检测对象属性的添加或删除**：由于劫持是发生在初始化阶段，后期直接 `obj.newKey = 1` 是非响应式的（需使用 `Vue.set`）。
2. **数组下标及长度变化的限制**：无法拦截 `arr[index] = val`。Vue 2 被迫通过重写数组的 7 个变异方法（push, pop 等）来变相实现，**object.definprotory 是能够监听到数组索引的变化的，vue不能监听是出于性能考量**
3. **性能负担**：初始化时需要**递归遍历**整个对象树，如果数据层级极深且量大，会导致严重的启动卡顿。

------

**Vue 3 响应式：基于 `Proxy`**

**核心机制：声明式劫持 + 副作用追踪**

Vue 3 放弃了对属性的劫持，转而对**整个对象**进行代理。

- **Proxy 代理**：使用原生 `Proxy` 拦截对象的各种操作（不仅是读写，还包括 `has`、`deleteProperty`、`ownKeys` 等）。
- **副作用函数（Effect）**：Vue 3 使用 `effect` 代替了 Watcher。当我们运行一个 effect（如组件渲染）时，它会将自己设为“当前激活的副作用”。
- **依赖图谱（Track & Trigger）**：
  - **Track（追踪）**：在 getter 中，将当前 `activeEffect` 存储到全局的 `targetMap` 中。结构为：`target -> key -> effects`。
  - **Trigger（触发）**：在 setter 中，根据 `target` 和 `key` 从 Map 中找到对应的 `effects` 集合并全部执行。

**缺点**：

- 兼容性没有vue2的好，没有办法polyfill



### vue3 proxy为什么要和reflect一起使用

一致性  ：在proxy的set方法中需要返回一个布尔类型的值，如果不适用reflect只能返回true或者false ，无法动态返回，使用reflct刚好可以根据是否设置成功返回true ，false 解决一致性的问题。

解决this指向的问题 ：在原始对象中使用this，会指向原始对象，在访问代理对象的时候无法进行依赖收集，需要使用reflct的recevier属性   将this改成代理后的对象。



### vue3响应式实现

```
// 1. 全局存储依赖的关系图
// 结构：target -> key -> Set(effects)
const targetMap = new WeakMap();
let activeEffect = null; // 当前正在运行的副作用函数

// 2. 追踪函数：收集依赖
function track(target, key) {
  if (!activeEffect) return; // 如果当前没有正在运行的 effect，直接返回

  let depsMap = targetMap.get(target);
  if (!depsMap) {
    targetMap.set(target, (depsMap = new Map()));
  }

  let dep = depsMap.get(key);
  if (!dep) {
    depsMap.set(key, (dep = new Set()));
  }

  // 把当前副作用函数存入集合
  dep.add(activeEffect);
}

// 3. 触发函数：派发更新
function trigger(target, key) {
  const depsMap = targetMap.get(target);
  if (!depsMap) return;

  const dep = depsMap.get(key);
  if (dep) {
    // 运行所有收集到的副作用函数
    dep.forEach(effect => effect());
  }
}

// 4. 定义 reactive
function reactive(target) {
  return new Proxy(target, {
    get(target, key, receiver) {
      const result = Reflect.get(target, key, receiver);
      track(target, key); // 【收集依赖】
      return result;
    },
    set(target, key, value, receiver) {
      const result = Reflect.set(target, key, value, receiver);
      trigger(target, key); // 【派发更新】
      return result;
    }
  });
}

// 5. 定义 effect（模拟组件渲染或侦听器）
function effect(fn) {
  activeEffect = fn;
  fn(); // 立即执行一次，触发 getter 从而调用 track
  activeEffect = null; // 执行完重置
}
```



### vue2响应式实现

```
// 1. 依赖收集器 Dep
class Dep {
  constructor() {
    this.subs = []; // 存储所有的 Watcher
  }
  // 添加订阅者
  addSub(sub) {
    this.subs.push(sub);
  }
  // 通知所有的订阅者更新
  notify() {
    this.subs.forEach(sub => sub.update());
  }
}

// 全局变量，用来暂存当前正在初始化的 Watcher
Dep.target = null;

// 2. 观察者 Watcher
class Watcher {
  constructor(vm, key, cb) {
    this.vm = vm;
    this.key = key;
    this.cb = cb;

    // 触发 getter，把自己收集到 Dep 里
    Dep.target = this;
    this.vm[key]; // 读取属性，触发 get 拦截
    Dep.target = null; // 收集完立即释放
  }
  update() {
    this.cb.call(this.vm, this.vm[this.key]);
  }
}

// 3. 核心：Observer 递归劫持
function observer(obj) {
  if (!obj || typeof obj !== 'object') return;

  Object.keys(obj).forEach(key => {
    defineReactive(obj, key, obj[key]);
  });
}

function defineReactive(obj, key, val) {
  // 递归处理嵌套对象
  observer(val);

  const dep = new Dep(); // 每个属性闭包一个 Dep 实例

  Object.defineProperty(obj, key, {
    enumerable: true,
    configurable: true,
    get() {
      // 如果此时有正在创建的 Watcher，则收集它
      if (Dep.target) {
        dep.addSub(Dep.target);
      }
      return val;
    },
    set(newVal) {
      if (newVal === val) return;
      val = newVal;
      observer(newVal); // 新值也需要劫持
      dep.notify(); // 【派发更新】
    }
  });
}
```
