### 区别

- **`ref`**: 对于基本类型，它使用 `Object.defineProperty` 的 `get/set`（类似于 Vue 2）；对于对象类型，它内部会自动调用 `reactive`。它会将数据封装在一个具有 `.value` 属性的对象中。

- **`reactive`**: 基于 **ES6 Proxy** 实现。它直接拦截整个对象的所有操作，不需要像 `ref` 那样多一层包裹。

| **特性**     | **ref**                                            | **reactive**                                   |
| ------------ | -------------------------------------------------- | ---------------------------------------------- |
| **数据类型** | **全能型**：支持基本类型（数字、字符串）和对象。   | **对象型**：只支持对象、数组、Map、Set。       |
| **访问方式** | JS 中需加 **`.value`**；模板中自动解包（不用加）。 | 直接访问属性（如 `state.count`）。             |
| **重新赋值** | **支持**：通过 `obj.value = {}` 替换整个对象。     | **不支持**：直接替换整个对象会**丢失响应式**。 |
| **解构支持** | 配合 `toRefs` 可解构。                             | 直接解构会**丢失响应式**。                     |
