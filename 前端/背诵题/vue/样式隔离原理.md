## 1. 为什么需要 scoped

在浏览器中 **CSS 默认是全局作用域**。

例如：

```
.title {
  color: red;
}
```

只要页面中存在：

```
<div class="title"></div>
```

样式就会生效。

因此在组件开发中容易出现：

- 样式污染
- 样式冲突
- 组件之间相互影响

在 Vue.js 中，可以使用 `scoped` 实现 **组件级样式隔离**。

------

# 2. scoped 的作用

```
<style scoped>
.title {
  color: red;
}
</style>
```

作用：

- 当前组件样式 **只作用于当前组件**
- 不会影响其他组件
- 默认不会作用到 **子组件内部 DOM**
- 也不会影响 **第三方组件**

------

# 3. scoped 的实现原理

`scoped` 本质是 **通过 PostCSS 在编译阶段重写 CSS 选择器实现的**。

实现步骤：

### ① 生成唯一的 scopeId

在组件编译时，Vue 会生成一个唯一标识：

```
data-v-xxxxxxx
```

例如：

```
data-v-3f2a8d1c
```

这个值一般根据：

- 文件路径
- 文件内容 hash

生成。

------

### ② 给组件 DOM 添加属性

Vue 在渲染 DOM 时会给 **当前组件所有元素**添加这个属性。

原始模板：

```
<template>
  <div class="box">
    <p class="text">hello</p>
  </div>
</template>
```

编译后 DOM：

```
<div class="box" data-v-3f2a8d1c>
  <p class="text" data-v-3f2a8d1c>hello</p>
</div>
```

------

### ③ 重写 CSS 选择器

原始 CSS：

```
.text {
  color: red;
}
```

编译后：

```
.text[data-v-3f2a8d1c] {
  color: red;
}
```

浏览器匹配规则：

```
class + 属性选择器
```

因此只会匹配当前组件。

------

# 4. scoped 的本质

scoped 本质就是：

```
CSS选择器 + 自定义属性选择器
```

例如：

```
[data-v-xxxx]
```

实现组件样式隔离。

------

# 5. scoped 对子组件的影响

默认情况下：

**父组件 scoped 样式不会影响子组件内部 DOM。**

例如：

父组件：

```
<style scoped>
.title {
  color: red;
}
</style>
```

子组件：

```
<div class="title"></div>
```

不会生效。

原因是：

父组件 CSS：

```
.title[data-v-parent]
```

子组件 DOM：

```
<div class="title" data-v-child>
```

属性不同，所以不会匹配。

------

# 6. 父组件可以影响子组件 root

Vue 有一个特殊处理：

**父组件的 scopeId 会添加到子组件的根节点上。**

例如：

```
<div class="child" data-v-child data-v-parent></div>
```

这样父组件可以控制：

- margin
- padding
- border
- layout

但不会影响子组件内部 DOM。

------

# 7. 样式穿透（v-deep）

如果需要让父组件样式影响子组件内部 DOM，可以使用 **深度选择器**。

Vue2：

```
::v-deep .title
```

Vue3 推荐：

```
:deep(.title)
```

例如：

```
<style scoped>
.parent :deep(.title) {
  color: red;
}
</style>
```

编译后：

```
.parent[data-v-parent] .title {
  color: red;
}
```

这样就可以影响子组件内部元素。

------

# 8. 其他深度选择器写法

历史兼容写法：

```
>>> .title
/deep/ .title
::v-deep .title
```

Vue3 推荐：

```
:deep(.title)
```

------

# 9. global 选择器

如果想写 **全局样式**，可以使用：

```
:global(.title) {
  color: red;
}
```

作用：

该样式不会添加 `data-v` 属性。

等价于：

```
普通全局CSS
```

------

# 10. scoped 的优缺点

### 优点

- 防止样式污染
- 组件样式独立
- 提高代码可维护性

### 缺点

- 选择器会增加属性匹配
- 修改子组件样式需要 deep
- 不是真正的 CSS 隔离（不是 Shadow DOM）

会编译成属性选择。





v-deep的本质就是在编译的时候把这个id去掉
