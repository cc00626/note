## watch和watchEffect的区别

| 区别     | watch            | watchEffect          |
| -------- | ---------------- | -------------------- |
| 依赖收集 | 手动指定         | 自动收集             |
| 执行时机 | 默认懒执行       | 立即执行             |
| 监听范围 | 指定变量         | 函数内所有响应式依赖 |
| 精确控制 | 强               | 弱                   |
| 常见用途 | 监听某个数据变化 | 自动响应副作用       |

> watch 需要显式指定监听的响应式数据，只有当指定数据变化时才会触发回调，并且可以获取 newValue 和 oldValue，适合需要精确控制监听源的场景。
> watchEffect 不需要指定监听源，而是在回调执行时自动收集依赖，只要回调中使用到的响应式数据变化就会重新执行副作用函数，并且会在创建时立即执行一次，因此更适合处理副作用逻辑。
