## 数据类型

六大基本数据类型：Number、Boolean、String、Undefined、Null、Symbol 

基本数据类型是指存放在栈中的简单数据段，它们是直接按值存放的，所以可以直接按值访问 



引用数据类型：Object 引用数据类型也叫对象数据类型，存放在堆(heap)内存中的数据，变量只是保存了对这块内存空间的引用



## 数据类型的判断

**typeof 判断数据类型**

用来判断基本数据类型，复杂数据类型除了function返回object，其中null返回object。

```javascript
typeof 123        // "number"
typeof "abc"      // "string"
typeof true       // "boolean"
typeof undefined  // "undefined"
typeof null       // "object"
typeof {}         // "object"
typeof []         // "object"
typeof function(){} // "function"
```

**instanceOf判断类型**

原理是根据原型链进行判断，判断左边的原型是否出现在右边的原型链进行判断。不能判断基本数据类型，基本数据类型需要使用String(),Number()进行包装，原型链不能够进行修改，需要进行包装。



不能够判断null和undefined

````javascript
function Test(){}
let test = new Test()
Test.prototype. proto = Array.prototype
console.log(test instanceof Array) // true
````

**Object.prototype.toString.call()判断数据类型**

在ES6之前，JS引擎会给js内部添加一个classs属性，表示每个对象的类型，比如function，object，array这个属性无法通过普通的js代码进行访问呢。在我们使用Object.prototype.toString.call()的时候就会调用这个属性。

在ES6之后，引入了一个Symbol.toStringTag全局属性。如果对象身上有这个属性。



**constructor判断数据类型**

利用原型身上的constructor属性来进行判断，在判断基本数据类型的时候，js会瞬时进行数据的包装然后进行数据的转换。



缺点：

1. null合undefined不能进行判断。
2. 原型链可能被重写，判断不准确

## JS数据类型的转换

1. **显示类型转换**

使用 `Number(),String(),boolean()` 对数据类型直接进行转换。

```javascript
//基本数据类型的时候
//字符串类型
Number('1') // 1
Number('1a') // NaN
Number('') // 0
//布尔类型
Number(true) // 1
Number(false) // 0
//null
Number(null) //0
//undefined
Number(undefined) //NaN



//数组
Number([]) // 0
//对象
Number({}) // NaN
//函数
Number(function(){})//NaN


//number
String(123) // '123'
//boolean
String(true) // 'true'
String(false) // 'false'
//null
String(null) // 'null'
//undefined
String(undefined) // 'undefined'


//数组
String([]) // ''
//对象
String({}) // "[object Object]"
//函数
String(function(){}) // 'function (){}'
```

boolean除`空字符串 null undefined 0 NaN false`下情况是false，剩下的情况都是true。    

**2. 隐式类型的转换**

**进行四则运算**

1. 使用 减、乘、除

   会把两边的变量转为数字，相当于对两边同时调用了`Number()`方法 

2. 使用 加

​	既能进行加法，也能进行字符串的相加

- 如果两边有一端是对象的时候，会先调用对象身上的`valueOf()`方法转换基本数据类型，如果方法返回的不是基本数据类型，则调用`toString()`方法
- 有一边是字符串的时候，就会尽量将两边同时转为字符串，然后进行字符串的相加。
- 不满足上面两点，则会转为Number类型，然后相加。

3. 使用 ==

​	如果类型相同，直接进行比较。除了 `**null == undefined**`：固定返回 `true`

​	如果类型不同

- 优先处理`!`, 将其转换为布尔值。
- 有一方是字符串，优先将字符串转换为数字

- 两边有一方是布尔值，会优先将布尔值转换成数字。

- 有一方是对象，会先调用对象身上的`valueOf()`方法转换基本数据类型，如果方法返回的不是基本数据类型，则调用`toString()`方法。



### 深拷贝和浅拷贝

**浅拷贝**

只拷贝对象的第一层属性

- 如果属性是基本数据类型，拷贝的是值
- 如果属性是引用类型，拷贝的是内存地址

常见的实现方式

**`Object.assign(target, ...sources)`**

**展开运算符 `...`** (ES6)

**数组方法**：`Array.prototype.slice()`、`Array.prototype.concat()`



**深拷贝**

递归拷贝对象的所有层级，新旧对象完全隔离，互不影响

常见的实现方式

**`JSON.parse(JSON.stringify(obj))` (快捷但不完美)**

这是开发中最常用的“歪门邪道”，但它有几个致命的 **面试必考缺点**：

1. **丢失属性**：会忽略 `undefined`、`Symbol`、`function`。
2. **数据失真**：`Date` 对象会变成字符串，`RegExp`、`Error` 会变成空对象，`NaN/Infinity` 会变成 `null`。
3. **循环引用**：如果对象里自己引用了自己，直接报错。
