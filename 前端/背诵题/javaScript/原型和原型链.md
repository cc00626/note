## 原型和原型链

**原型**

在创建一个函数的时候，在函数的身上有一个protoType属性，指向**原型对象**。原型对象中有一个constractor属性指向构造函数。

通过这个函数创建出来的实例身上有一个proto属性，指向原型对象，同时也有一个constractor指向构造函数。称为**原型**



**原型链**

当某个原型对象是另一个类型的实例的时候，该原型对象内部Proto指针会指向另一个类型的原型对象 (所有对象都是Object的实例)，如此层层递进就构成了实例和原型的链条，这就是原型链的概念。



## 继承方式

### 1. 原型链继承

```javascript
function Person() {
  this.name = "juege"
}

Person.prototype.eating = function () {
  console.log("eating")
}

function Student() {}
Student.prototype = new Person()

// 前端实现类的过程中都没有传递参数。
// 多个实例对引用类型的操作会被篡改。
// 打印子类实例，子类继承到的属性是永远看不见的。
```

### 2. 构造函数继承

```javascript
function Student(name, number) {
  Person.call(this, name)
  this.number = number
}

Student.prototype = new Student()
// 缺点:父类构造函数被调用了两次，给子类元素上造成了一些没必要存在的元素。
```

### 3. 组合寄生继承

```javascript
// 整体思路
// 1.盗用构造函数 .call
// 2.原型赋值   Object.create()
// 3.修改constructor

function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.running = function () {
  console.log("我会跑");
};

function Student(name, age, Number) {
  Person.call(this, name, age); //核心代码
  this.Number = Number;
}

inheritPrototype(Student, Person);

//原型赋值+修改constructor
function inheritPrototype(children, parent) {
  children.prototype = Object.create(parent.prototype);
  // children.prototype.constructor = children;
  Object.defineProperty(children.prototype, "constructor", {
    value: children,
    enumerable: false,
    configurable: true,
    writable: true,
  });
}
```
