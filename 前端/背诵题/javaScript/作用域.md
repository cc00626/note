### js中的作用域

| **作用域类型** | **描述**                                    | **声明方式**                                    |
| -------------- | ------------------------------------------- | ----------------------------------------------- |
| **全局作用域** | 整个脚本文件的任何地方都能访问。            | 在函数外部声明，或直接挂在 `window/global` 上。 |
| **函数作用域** | 变量只在函数内部可见，外部无法访问。        | `var`、`let`、`const` 在函数内声明时。          |
| **块级作用域** | 由一对花括号 `{}` 包裹的区域。              | **只有** `let` 和 `const` 遵循此规则。          |
| **模块作用域** | 在 ES Module 模式下，变量仅在当前文件可见。 | `export` 和 `import` 控制其流转。               |

JavaScript 使用的是“静态作用域”（词法作用域）

### 作用域链的查找

当代码执行遇到一个变量时，JS 引擎会触发“向上查找”机制：

1. **当前作用域**：先在当前 `{}` 或函数内找。
2. **父级作用域**：找不到就去定义它的上一层找。
3. **全局作用域**：一直找到 `window`（浏览器）或 `global`（Node.js）。
4. **报错**：如果直到全局都找不到，抛出 `ReferenceError`。

### let、const和var的区别

### 1. 作用域 (Scope)

- **`var`：函数级作用域**。它只认函数，不认代码块（如 `if` 或 `for`）。如果你在 `if` 块里定义了 `var`，在块外面依然能访问它。
- **`let` / `const`：块级作用域**。它们只在最近的一对花括号 `{}` 内有效。这完美解决了循环变量泄露、全局变量污染的问题。

------

### 2. 变量提升 (Hoisting) 与 暂时性死区 (TDZ)

- **`var`：会提升**。JavaScript 会把声明移到作用域顶部，并初始化为 `undefined`。所以你在声明前打印它，不会报错，而是输出 `undefined`。
- **`let` / `const`：存在暂时性死区 (TDZ)**。虽然它们在语法上也会被提升，但在执行到声明语句之前，它们是**不可访问**的。如果在声明前调用，会直接抛出 `ReferenceError`。

------

### 3. 重复声明 (Redeclaration)

- **`var`：允许重复声明**。你可以写无数个 `var a = 1; var a = 2;`，系统不会报错，这往往会导致无意间的变量覆盖。
- **`let` / `const`：禁止重复声明**。在同一个作用域内，如果重复声明同一个变量，编译器会直接报错。

------

### 4. 修改与初始化 (Mutation & Initialization)

| **特性**       | **var**            | **let**            | **const**                |
| -------------- | ------------------ | ------------------ | ------------------------ |
| **可重新赋值** | ✅ 可以             | ✅ 可以             | ❌ 不可以                 |
| **必须初始化** | ❌ 默认 `undefined` | ❌ 默认 `undefined` | ✅ 必须在声明时赋值       |
| **内存本质**   | 指向可变的地址     | 指向可变的地址     | **指向的内存地址不可变** |

> **注意 `const` 的误区：** `const` 保证的是变量指向的**地址**不变。如果你用 `const` 定义了一个对象或数组，你可以修改对象内部的属性或数组的元素，但不能给这个变量重新赋值一个新的对象。
