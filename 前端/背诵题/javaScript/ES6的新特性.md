### Map和Set

**set集合**

Map

| **特性**     | **Object**                           | **Map**                          |
| ------------ | ------------------------------------ | -------------------------------- |
| **键的类型** | 只能是 **String** 或 **Symbol**      | **任意类型**（对象、数字等均可） |
| **键的顺序** | 无序（虽然现代引擎有优化，但不保证） | **有序**（按插入顺序排列）       |
| **大小获取** | 需要手动遍历 `Object.keys().length`  | 直接访问 `.size`                 |
| **性能**     | 频繁增删时性能一般                   | 频繁增删时**性能更好**           |



### Map和WeakMap的区别

| **特性**          | **Map**                                    | **WeakMap**                              |
| ----------------- | ------------------------------------------ | ---------------------------------------- |
| **键的类型**      | 任意类型（数字、字符串、对象等）           | **必须是对象**（或 Symbol，视版本而定）  |
| **垃圾回收 (GC)** | **强引用**：键不会被垃圾回收               | **弱引用**：键如果没有其他引用，会被回收 |
| **可遍历性**      | 可以遍历 (`keys()`, `values()`, `forEach`) | **不可遍历**，没有 `size` 属性           |
| **主要用途**      | 存储需要频繁查改的数据                     | 存储与对象关联的元数据，防内存泄漏       |
